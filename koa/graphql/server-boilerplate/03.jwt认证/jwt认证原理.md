* jwt 由3部分构成, 第1部分是加密算法   
```
     {
         alg: "H256" // 哈希256算法
         typ: "jwt" // 加密类型
     }
     // 第2部分, 给用户的信息
     {
         id: "123344",
         "name": "abc",
         "exp": 1h // 过期时间
     }
     // 第3部分
     {
        加密的内容
     }
```
```
 当服务器拿到 token 后, 第1, 2 部分相当明文, 第3部分反解不了, 
 只能通过第1部分+第2部分重新拼接,
 再重新进行 H256进行加密 = 重新生成的密文A
 密文在用 base64进行解码: 新的密文 = base64(重新生成的密文A)
 原来的密文 = base64(第3段的密文)
 比较新的密文和原来的密文.看看是否相等, 如果相等, 则 token 没有修改过, 验证通过
```
